别再手动刷公众号推文了：24小时内 AI 公众号推文自动去重归档，飞书群里直接看头条与摘要。我用 n8n + 飞书 + Agent 搭了个「AI公众号推文情报台」：采集/清洗/摘要/选编/推送/回写闭环全打通

## 1、案例介绍

本期视频为大家分享的是践行“**高效的工具(n8n+Agent+Cursor+飞书)” + “标准化的流程(SOP)” + “严格的监督反馈机制（人工）**”三者结合，构建一个能替代传统方式的增长引擎系列案例       

本次要分享的应用案例是**AI公众号推文采集和智能推送支持平台**，主要实现每天自动采集并归纳 AI 公众号推文，智能提炼重点内容后推送给我；我在飞书消息里即可实时预览头条/深度/快讯的关键信息与摘要，无需再手动筛选整理            

涉及到的工作流配置文件、操作说明文档等全部资料都是开源分享给大家的，大家可以在本期视频置顶评论中获取免费资料链接进行下载                      

本期用例的核心功能包含：    

- **RSS 源管理与读取**：从飞书多维表格读取“已启用”的 RSS 源列表，并逐个拉取最新内容
- **时效筛选**：仅保留 24 小时内发布的公众号推文条目，减少噪声与过期内容
- **去重入库**：与多维表格里近 30 天/当月已入库记录按 标题 + 链接 去重，避免重复采集与重复推送
- **内容清洗**：对 RSS 返回内容做清洗（如 Markdown/HTML → 纯文本、字段规整），为后续 AI 处理提供干净输入
- **AI 摘要与打标**：对每条公众号推文生成结构化摘要并输出机器标签
- **写入选题池/公众号推文池**：将标题、摘要、原文链接、来源信息、发布时间、机器标签等写入飞书多维表格，并默认标记“暂未推荐”
- **AI 智能选编**：每天定时拉取当日新增公众号推文，让 AI 按策略归类为 头条 / 深度 / 快讯，并给出入选理由与统计
- **飞书群富文本推送**：把 AI 归类结果渲染成飞书 post 富文本消息，推送到指定群聊
- **回写推荐类型**：将被选中的多维表格记录回写“推荐类型”（头条/深度/快讯），形成可追踪闭环

## 2、整体解决方案架构

本方案采用“自动采集 → 智能清洗 → AI摘要 → 结构入库 → 智能推送”的闭环链路，核心技术栈为：
- **n8n 自动化工作流**：负责整个数据流的串联、定时调度与异常处理
- **飞书多维表格**：作为灵活的公众号推文归档、标签与推送策略配置中台
- **Agent/LLM/正则代码**：实现内容标准化、摘要生成与文本提取
- **飞书消息API**：用于自动化推送内容、交互与反馈

集成流程如下：

```
定时触发（如每天7:30）  
    ↓  
从飞书多维表拉取 RSS 源  
    ↓  
循环读取各 RSS 源条目  
    ↓  
24小时+去重筛选  
    ↓  
内容清洗（Markdown/HTML转纯文本）  
    ↓  
AI 生成摘要+标签  
    ↓  
写入飞书多维表/更新来源库  
    ↓  
AI归类分级（头条/深度/快讯等）  
    ↓  
飞书群自动推送富文本  
    ↓  
回写推荐标签
```

## 3、主要节点与配置说明

### 3.1 多维表格节点

- **读取“已启用”RSS源**：配置好 API Token 和表格ID，拉取启用状态为true的源列表。
- **写入公众号推文数据**：通过 HTTP Request，将结构化数据（标题、摘要、推荐类型等字段）写回多维表，实现动态入库与更新。

### 3.2 RSS 读取和处理节点

- **循环读取**：Loop Over Items 节点，对多维表返回的每个源依次读取。
- **RSS Read**：解析 RSS 源，提取关键信息（title、link、content、pubDate、source）。
- **去重处理**：对已采集的内容按“标题+链接”字段与表格历史记录去重，仅保留新的有效内容。

### 3.3 内容清洗

- **Code 节点自定义清洗**：使用正则/JavaScript自定义代码，完成 Markdown/HTML 转纯文本、字段归一化，详见 `【01_子工作流】AI公众号推文采集工作流.json` 文件说明。

### 3.4 AI摘要节点

- **LLM Orchestration/Agent节点**：自动调用大模型接口，为每条公众号推文生成高质量摘要、打标签。可自定义prompt区分不同内容类别（快讯/深度/头条）。

### 3.5 智能推送

- **归类算法与自动推送**：再用 Code/AI 节点实现自动分组推荐；消息通过“飞书机器人”发送富文本格式消息，支持群聊/多维表格内容回写。

## 5、快速上手指引（简要版）

1. **准备环境**：本地或云端部署 n8n（推荐 Docker），注册飞书开发者帐号、创建多维表格、配置机器人，并获取 Token。
2. **导入工作流**：将本项目 `工作流配置文件/` 下的 JSON 通过 n8n Control Panel 导入。
3. **配置参数**：完善环境变量、API Token、多维表格ID等必填项。
4. **补充 RSS源表格**：在飞书多维表“RSS源列表”工作表，维护需要采集的公众号推文源条目。
5. **定时触发/测试运行**：可设置定时，也可手动触发主工作流，观察输出数据与推送结果。

可结合`操作说明/`内更详细文档查阅，提升部署复现效率。
